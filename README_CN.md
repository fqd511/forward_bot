# Telegram 消息自动转发工具 (Forward Bot) - 中文说明

本工具是一个轻量、高效的 Telegram 消息自动化转发解决方案，支持**手动命令行执行**与**常驻服务**两种运行模式。

---

## 🛠️ 运行模式说明

### 1. 命令行手动转发模式 (核心推荐 ✨)
*   **入口脚本**：`forward-cli.js` (运行命令: `npm run forward`)
*   **工作机制**：使用 **用户身份（Userbot - 基于 GramJS）** 模拟真人操作进行转发。
*   **适用场景**：手动、大批量、历史消息的定向转发、清理与合并。
*   **核心特色**：
    *   **破除 Bot 限制**
    *   **成组消息合并**
    *   **高弹性过滤**：支持排除/包含正则表达式，以及对图片/视频/动图/音频/语音/贴纸等 8 类媒体的精准过滤。
    *   **多账号切换**：支持登录和保存多个用户 Session，执行任务前可自由选择以哪一个账号身份执行。
    *   **拟人防封保护**：支持发送间隔的随机时间抖动 (Jitter)，最大化保障账号安全。

### 2. 自动服务常驻模式 (开发中 🚧)
*   **入口脚本**：`index.js` (运行命令: `npm run dev`)
*   **工作机制**：使用 **Bot 身份（基于 Telegraf）** 监听实时新消息事件进行转发。

---

## 🚀 命令行手动转发模式使用步骤

1.  **复制配置并填入凭证**：
    *   复制 `.env.sample` 命名为 `.env`，填入 `DEST_CHANNEL`（目标频道）和你的 `API_ID` 与 `API_HASH`（可在 [my.telegram.org](https://my.telegram.org) 免费创建）。
    *   如果在国内运行，请在 `.env` 中配置 `TELEGRAM_PROXY`（Socks5 协议，如 `socks5://127.0.0.1:7890`）。

2.  **自定义过滤规则**：
    *   根据需要修改 `forward-config.json` 中的过滤正则和媒体类型开关（该文件内含有详细的使用说明和示例）。

3.  **运行程序**：
    ```bash
    npm run forward
    ```
    *   **账号选择**：首次启动需选择登录新账号并输入验证码进行认证。后续启动将列出所有已登录的账号，供你直接选择使用。
    *   **范围输入**：输入起始消息链接（必填）和结束消息链接（可选，留空则一直转发到最新可用消息）。

---

## 📂 项目结构
*   `forward-cli.js` — 命令行手动转发核心脚本
*   `forward-config.json` — 消息过滤、延时与隐藏来源等参数配置文件
*   `index.js` — 常驻服务自动转发入口（未完成）
*   `.sessions/` — 存放已登录的多账号 Session 凭证（已被 `.gitignore` 排除，安全保密）
